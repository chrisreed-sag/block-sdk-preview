Constant detail

DEFAULT_TIMEOUT_SECS

constant float DEFAULT_TIMEOUT_SECS := 10.0
Default timeout for simple request-responses.

Timeout in seconds applied to request-responses, for example in requests to Cumulocity.

LOGGING_THROTTLE_SECS

constant float LOGGING_THROTTLE_SECS := 1.0
Logging throttling in seconds.

Interval used for periodic logging, such as reporting changes in the number of dropped events by Input blocks (for events delayed by more than TIMEDELAY_SECS). Log statements will not appear more frequently than this period.

MINIMUM_WAIT_TIME_SECS

constant float MINIMUM_WAIT_TIME_SECS := 0.1
Minimum wait time in seconds.

Some blocks can generate output automatically, based on the rate of change of the output. This sets a lower limit on the time between outputs, in seconds.

TIMEDELAY_SECS

constant float TIMEDELAY_SECS := 1.0
The maximum delay in seconds before Input blocks consider an event to be old.

Input blocks re-order input events, which is achieved by not processing them until TIMEDELAY_SECS after the timestamp in the object, thus delaying processing by this amount. If the event is delayed in its delivery to the correlator, such that the delay is greater than or equal to this value, then the input is considered old and will be dropped and not processed.
